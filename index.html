Introduction

Welcome. This is a guide on how to complete the tribute page responsive web design project from FreeCodeCamp. Before we can get started, here is a list of things we need:

  • A text editor to save your code. This is not needed if you are working in CodePen.
  • A web browser
  • Basic knowledge of HTML
  • Basic knowledge of CSS

CodePen vs Locally

You can do this project directly on CodePen by forking the pen provided by FreeCodeCamp and save everything there. If you want to work locally and have your code uploaded to an environment like Github then you need to write your code in a text editor and save the code as an HTML file. You then use your web browser to see the result. If you are working locally, you will need to add the following elements:

  • <!DOCTYPE html> is not an element but a declaration at the start of every html document. You can inspect a webpage to see for yourself.
  • <html> between the opening and closing tag sits the rest of our html code.
  • <head> for the next two elements.
  • <meta> for metadata. We will use it to have our page be responsive by adding in the following code
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • <style> for CSS code if you choose to do it in the same file as your html. If not, then you do not include this element. Instead you include a <link> element to link to your external CSS file.
  • <body> is where the text and images go, the things we see when we view a webpage.
  • <script src="https://cdn.freecodecamp.org/testable-projects-fcc/v1/bundle.js"><script> in the body element to test your page.

User Story #1

We need to have an element that holds all other elements. We need to give that element an id attribute with the value "main". We can use the code<main> element and give it the attribute code<id="main">. So what we have is opening tag with attribute <main id="main"> and closing tag </main> that all our other elements will be contained in.

User Story #2

We want an element with an id attribute with the value "title" whose content will tell readers immediately at a glance who or what is the tribute page dedicated to. We can use a header element such as <h1>. It will look something like this <h1 id="title">Your title</h1>

User Story #3

We will use the <figure> and give it the id attribute "img-div".
<figure id="img-div">
</figure>

User Story #4

Now it is time to actually add the image. We will have an <img> element within the <figure> element and we need to give it an id attribute with the value "image". Find an image that relates to your topic. I recommend using an image that is Creative Commons licensed.
<img id="image" src="your image link" alt="your alt text">

User Story #5

Next, we have to give our image a caption. We can use the <figcaption> element to add a caption and give the element an id attribute with the value "img-caption". It will be within our <figure> element together with our <img> element. <figcaption id="img-caption">Your image caption</figcaption>

User Story #6

We need an element whose content will include facts on our tribute page topic and we need to give it an attribute id with the value "tribute-info". We can create a <div> element that will hold other elements like <p> or <ul> elements that will subsequently have the factoids as content.

User Story #7

Next, we need to provide a link to an external website that gives more information on the topic of our tribute page. We will use an <a> with an id attribute whose value is "tribute-link". If we want the link to open in a new tab we give the element the <target> attribute with the value _blank.
<a id="tribute-link" href="external site link" target="_blank">Your hyperlink text</a>

User Story #8

We want our image to be responsive. It should resize according to the width of its parent element and its width should not exceed its original size. Here we get to use CSS to style our <img>. We can select our element by its id attribute. We use the pound/hash symbol # and the id attribute value. #image { your css goes here } Your CSS goes between brackets. Style away.

<img> elements are by default inline-block elements. We want it to be display: block;. Next, we give it height: auto; and max-width: 100%; to allow the image to become smaller or larger depending on the viewport but never exceeding its original size.

User Story #9

The last test is for the image to be centered within its container. We will style our image with margin: auto; which can automatically center block elements horizontally.
